MASALAH #1: Schedule Service Stops When GUI Closes
───────────────────────────────────────────────────

FILE: app/main.py
LINE: 266-274 (OLD CODE)

KODE LAMA (SALAH):
```python
# Cleanup on exit - always stop GUI services
logger.info("Shutting down GUI services")
schedule_service.stop()  ← INI MASALAHNYA!
retention_service.stop()
logger.info("GUI application closed")
```

EFEK:
1. User buka GUI → Schedule service start
2. User tutup GUI → Schedule service STOP!
3. Schedules TIDAK jalan di background
4. Backup jam 15:56 TIDAK execute karena service sudah stop

KODE BARU (BENAR):
```python
# Cleanup on exit
if service_running:
    # Task Scheduler background service is running
    # Safe to stop GUI services
    logger.info("Shutting down GUI services")
    schedule_service.stop()
    retention_service.stop()
else:
    # No background service - warn user and keep GUI services running
    logger.warning("⚠️ WARNING: Task Scheduler not configured!")
    # DON'T stop services - let them continue
    logger.info("GUI services kept alive for schedule execution")
```

SOLUSI:
✅ Jika Task Scheduler configured: stop GUI services (safe)
✅ Jika TIDAK configured: KEEP services alive + warning
✅ Schedules AKAN jalan bahkan setelah GUI ditutup

ENCRYPTION METHOD:
──────────────────
```python
# Save (encrypt)
key = b'AlliedTelesisBackupGUI2025'
encoded = base64.b64encode(passphrase.encode('utf-8'))
encrypted = XOR(encoded, key)
save to file: data/.gui_passphrase

# Load (decrypt)
encrypted = read from file
decrypted = XOR(encrypted, key)
passphrase = base64.b64decode(decrypted).decode('utf-8')
```
